# otcc init Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现 `otcc init` 命令，在当前项目的 `CLAUDE.md` 顶部创建或更新 `<otcc-role>` 提示块，并从包内模板文件读取内容。

**Architecture:** 新增一个 `src/commands/init.ts` 作为 oclif 命令入口，把 CLI 解析与输出留在命令层；新增 `src/core/init/claude.ts` 负责模板读取、block 生成、文本替换和文件写入决策。测试分成两层：纯逻辑测试验证 block 注入行为，命令级测试验证 `otcc init` 对真实工作目录文件的操作结果。

**Tech Stack:** TypeScript, Bun, oclif, node:fs/promises, bun test

## File Structure

### New files

- `templates/role-priority-prompt.md` — 运行时模板文件，随 npm 包发布，作为 `<otcc-role>` 的正文来源
- `src/core/init/claude.ts` — 模板定位、block 生成、文本替换、文件写入与结果类型定义
- `src/commands/init.ts` — `otcc init` 命令入口
- `test/core/init/claude.test.ts` — 纯逻辑与文件级行为测试
- `test/commands/init.test.ts` — 命令级测试，验证当前目录 `CLAUDE.md` 的创建、插入、替换与失败行为

### Modified files

- `package.json` — 增加 `templates/` 到 `files`，增加 `test` script
- `tsconfig.json` — 让测试目录也能参与类型检查（如果你决定对测试文件执行 `tsc --noEmit`）
- `docs/role-priority-prompt.md` — 保留为说明文档；不作为运行时依赖

## Implementation Notes

- block 标签固定为 `<otcc-role>`，不做配置化
- 目标文件固定为当前工作目录的 `CLAUDE.md`
- 若已有完整 block，则替换首个完整 block
- 若检测到损坏 block（只有开始标签或只有结束标签），直接抛错
- 重复执行必须幂等
- 模板读取应基于当前模块文件位置回溯到包根目录，而不是基于 `process.cwd()`

# 角色优先提示词

在处理任何任务前，你必须先执行“角色优先”检查流程。

## 角色优先规则

1. 当收到用户任务时，先理解任务目标、范围和类型。
2. 在开始执行任务前，优先检查当前是否已有合适的角色（role）可用于处理该任务。
3. 如果存在合适角色：
   - 选择最匹配的角色优先执行；
   - 如有多个候选角色，优先选择职责最贴近、范围最明确的那个；
   - 不要无故跳过角色选择。
4. 如果不存在合适角色：
   - 不要直接继续执行任务；
   - 必须先询问用户，并明确给出以下 3 个选项：

### 请选择如何继续

1. 跳过角色检查，直接继续执行当前任务
2. 立即创建角色
3. 等我创建好角色后再继续

## 选项处理规则

### 如果用户选择 1：跳过角色检查

- 直接继续执行当前任务；
- 后续不再重复追问是否创建角色，除非任务范围发生明显变化。

### 如果用户选择 2：立即创建角色

- 调用 `otcc:role-creator` 来创建角色；
- 创建角色时，基于当前任务生成合理的角色定位、职责边界和适用范围；
- 角色创建完成后，继续当前任务；
- 优先使用刚创建的角色处理后续工作。

### 如果用户选择 3：等用户创建好角色后再继续

- 立即停止当前任务执行；
- 明确告知用户：当前任务已暂停，等待用户完成角色创建后再继续；
- 不要继续分析、规划、编码或修改文件。

## 重要约束

- “角色优先检查”必须发生在任务执行前，而不是执行中途。
- 如果任务非常简单，也仍然要先判断是否已有合适角色。
- 如果用户已经明确指定某个角色，则直接使用该角色，不再重复询问。
- 如果没有合适角色，必须严格使用上述 3 个选项，不要自行扩展成别的选项。
- 在用户未选择前，不得擅自决定创建角色或跳过角色流程。

## 交互风格

当没有合适角色时，请使用类似下面的话术向用户提问：

“当前没有找到适合处理这个任务的角色。请选择下一步操作：

1. 跳过角色检查，直接继续执行
2. 立即创建角色
3. 等你创建好角色后再继续”

如果用户选择 3，请使用类似下面的话术结束当前流程：

“好的，当前任务已暂停。你创建好角色后告诉我，我再继续执行。”

```ts
import { access, readFile, writeFile } from "node:fs/promises";
import { dirname, resolve } from "node:path";
import { fileURLToPath } from "node:url";

export type InitAction = "created" | "inserted" | "updated";

export interface ApplyOtccRoleBlockResult {
  action: InitAction;
  content: string;
}

export interface InitClaudeResult {
  action: InitAction;
  filePath: string;
}

const START_TAG = "<otcc-role>";
const END_TAG = "</otcc-role>";

export function buildOtccRoleBlock(template: string): string {
  const normalized = template.replace(/\s+$/u, "");
  return `${START_TAG}\n${normalized}\n${END_TAG}`;
}

export function applyOtccRoleBlock(
  current: string | null,
  block: string,
): ApplyOtccRoleBlockResult {
  if (current === null) {
    return {
      action: "created",
      content: `${block}\n`,
    };
  }

  const hasStart = current.includes(START_TAG);
  const hasEnd = current.includes(END_TAG);

  if (hasStart !== hasEnd) {
    throw new Error("检测到损坏的 <otcc-role> block");
  }

  if (hasStart && hasEnd) {
    const pattern = /<otcc-role>[\s\S]*?<\/otcc-role>/u;
    return {
      action: "updated",
      content: current.replace(pattern, block),
    };
  }

  return {
    action: "inserted",
    content: `${block}\n\n${current.replace(/^\n+/u, "")}`,
  };
}

export function resolveTemplatePath(): string {
  const currentDir = dirname(fileURLToPath(import.meta.url));
  return resolve(currentDir, "../../../templates/role-priority-prompt.md");
}

export async function readRolePriorityTemplate(): Promise<string> {
  return readFile(resolveTemplatePath(), "utf-8");
}

export async function initClaudeMd(cwd: string): Promise<InitClaudeResult> {
  const filePath = resolve(cwd, "CLAUDE.md");
  const template = await readRolePriorityTemplate();
  const block = buildOtccRoleBlock(template);

  const exists = await access(filePath)
    .then(() => true)
    .catch(() => false);
  const current = exists ? await readFile(filePath, "utf-8") : null;
  const result = applyOtccRoleBlock(current, block);

  await writeFile(filePath, result.content, "utf-8");

  return {
    action: result.action,
    filePath,
  };
}
```

import { mkdir, mkdtemp, readFile, rm, writeFile } from "node:fs/promises";
import { tmpdir } from "node:os";
import { dirname, resolve } from "node:path";

const projectRoot = resolve(import.meta.dir, "../..");

async function runInit(cwd: string) {
  return Bun.$`bun run ${projectRoot}/src/index.ts init`.cwd(cwd).quiet();
}

describe("otcc init command", () => {
  let workspace: string;

  beforeEach(async () => {
    workspace = await mkdtemp(resolve(tmpdir(), "otcc-init-"));
  });

  afterEach(async () => {
    await rm(workspace, { recursive: true, force: true });
  });

  test("creates CLAUDE.md when missing", async () => {
    await runInit(workspace);

    const content = await readFile(resolve(workspace, "CLAUDE.md"), "utf-8");
    expect(content).toContain("<otcc-role>");
    expect(content).toContain("# 角色优先提示词");
  });

  test("inserts block at top when CLAUDE.md already exists", async () => {
    await writeFile(
      resolve(workspace, "CLAUDE.md"),
      "# Existing\n\nHello\n",
      "utf-8",
    );

    await runInit(workspace);

    const content = await readFile(resolve(workspace, "CLAUDE.md"), "utf-8");
    expect(content.startsWith("<otcc-role>\n")).toBe(true);
    expect(content).toContain("\n\n# Existing\n\nHello\n");
  });

  test("replaces existing otcc-role block", async () => {
    await writeFile(
      resolve(workspace, "CLAUDE.md"),
      "<otcc-role>\nold\n</otcc-role>\n\n# Existing\n",
      "utf-8",
    );

    await runInit(workspace);

    const content = await readFile(resolve(workspace, "CLAUDE.md"), "utf-8");
    expect(content).toContain("# 角色优先提示词");
    expect(content).not.toContain("<otcc-role>\nold\n</otcc-role>");

  test("fails on broken otcc-role block", async () => {
    await writeFile(
      resolve(workspace, "CLAUDE.md"),
      "<otcc-role>\nold\n",
      "utf-8",
    );

    await expect(runInit(workspace)).rejects.toThrow();
  });

```ts
import { Command } from "@oclif/core";
import { initClaudeMd } from "../core/init/claude";

export default class Init extends Command {
  static description = "初始化当前项目的 CLAUDE.md 角色提示块";

  static examples = ["$ otcc init"];

  async run(): Promise<void> {
    const result = await initClaudeMd(process.cwd());

    if (result.action === "created") {
      this.log("✅ 已创建 CLAUDE.md");
      return;
    }

    if (result.action === "inserted") {
      this.log("✅ 已在 CLAUDE.md 顶部插入 <otcc-role>");
      return;
    }

    this.log("✅ 已更新现有 <otcc-role>");
  }
}
```
